================================================================================
                        HEALTH ENDPOINTS EXPOSED
================================================================================

Per Service (Order: 8080, Product: 8081, Payment: 8082):

  1. /api/actuator/health
     └── Full health report with all components
     └── HTTP 200 (UP) or HTTP 503 (DOWN)

  2. /api/actuator/health/live
     └── Liveness probe - is service running?
     └── HTTP 200 (UP) or HTTP 503 (DOWN)

  3. /api/actuator/health/ready
     └── Readiness probe - is service ready for traffic?
     └── HTTP 200 (UP) or HTTP 503 (DOWN)

  4. /api/actuator/info
     └── Service information and metadata

  5. /api/actuator/metrics
     └── Service metrics and performance data

================================================================================
                         CUSTOM INDICATORS
================================================================================

1. KafkaHealthIndicator
   📌 Component: kafkaHealth
   📍 Action: Sends test message to Kafka
   ✓ UP: Message sent successfully
   ✗ DOWN: Broker unreachable or timeout
   ⏱ Timeout: 5 seconds
   📊 Logging: INFO (success), ERROR (failure)

2. DatabaseHealthIndicator
   📌 Component: databaseHealth
   📍 Action: Attempts to get connection from pool
   ✓ UP: Connection available and active
   ✗ DOWN: Connection unavailable or closed
   ⏱ Timeout: Connection pool timeout (default 30s)
   📊 Logging: INFO (success), ERROR (failure)

================================================================================
                         STARTUP PROCESS
================================================================================

1. START INFRASTRUCTURE (Docker)
   $ docker-compose up -d
   ├── Kafka (port 9092)
   ├── PostgreSQL Order (port 5432)
   ├── PostgreSQL Product (port 5433)
   └── PostgreSQL Payment (port 5434)

2. BUILD SERVICES
   $ mvn clean install -DskipTests

3. START SERVICES (3 terminals)
   Terminal 1: cd order-service && mvn spring-boot:run (port 8080)
   Terminal 2: cd product-service && mvn spring-boot:run (port 8081)
   Terminal 3: cd payment-service && mvn spring-boot:run (port 8082)

4. VERIFY HEALTH
   $ bash verify-health.sh
   
   Expected Output:
   ✓ Order Service is HEALTHY
   ✓ Product Service is HEALTHY
   ✓ Payment Service is HEALTHY
   All services are HEALTHY!

================================================================================
                          TROUBLESHOOTING
================================================================================

ISSUE: Service won't start
  ├── Check Docker: docker ps
  ├── Check Kafka: docker logs kafka-kraft
  └── Check DB: docker logs postgres-order

ISSUE: Health endpoint returns 503
  ├── View full response: curl http://localhost:8080/api/actuator/health | jq
  ├── Check Kafka logs: docker logs kafka-kraft
  └── Check DB logs: docker logs postgres-order

ISSUE: Kafka health always DOWN
  ├── Verify running: docker ps | grep kafka
  └── Test connection: docker exec kafka-kraft kafka-broker-api-versions.sh...

ISSUE: Database health always DOWN
  ├── Verify running: docker ps | grep postgres
  └── Test connection: psql -h localhost -p 5432 -U order_user -d order_db...
